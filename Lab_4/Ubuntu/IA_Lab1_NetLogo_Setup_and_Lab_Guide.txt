Important:

The PDF does NOT require Ubuntu 14.04 or 16.04 specifically.
It focuses on learning the NetLogo interface, basic terminology, and
simple experiments in a NetLogo model.

==================================================
PART 1 - INSTALL NETLOGO
==================================================

Ubuntu

1) Download NetLogo from the official website in your browser
2) Open a terminal
3) Extract it into a tools folder in your home directory

cd ~
mkdir -p tools
cd tools
tar -xf ~/Downloads/NetLogo-*.tgz
ls

4) Start NetLogo

cd ~/tools/NetLogo*/
./NetLogo

Expected:
- NetLogo opens
- you can see the Interface, Info, and Code tabs

Windows

1) Download NetLogo
2) Install it normally
3) Open NetLogo

==================================================
PART 3 - UNDERSTAND THE INTERFACE
==================================================

The PDF explains these important NetLogo elements:

Environment
- The black square on the right is the environment, the graphical
  representation of the model

Toolbar
- At the top, the toolbar lets you:
  - modify the interface
  - change execution speed
  - change display mode
  - open environment settings

Settings
- The Settings window lets you change:
  - coordinate system origin
  - world size
  - topology
  - patch size

Command line
- You can send commands to:
  - turtles
  - patches
  - links
  - observer

Agent types
- turtles = moving agents
- patches = fixed grid cells
- links = connections between turtles
- observer = the one that runs and observes the model

Ticks
- Time is measured in discrete steps called ticks

Buttons
- setup prepares the initial state
- go runs the model
- a forever button keeps running until you stop it

Parameters
- The interface can include:
  - switch
  - slider
  - chooser
  - input

Tracking outputs
- Interface elements for results:
  - monitor
  - plot
  - output

Code tab
- In the Code section, you can:
  - read and edit the program
  - use Find
  - use Procedures
  - check syntax with Check

==================================================
PART 4 - DO THE LAB STEP BY STEP
==================================================

STEP 1 - START THE MODEL

Press the Setup button.

Expected:
- the environment is populated with turtles according to the selected number

Then press the Go button.

Expected:
- the model starts running
- turtles begin to move according to the programmed behavior

STEP 2 - EXPERIMENT WITH PARAMETERS

Change the values of the available sliders, switches, or choosers.

Observe:
- which parameters affect initialization
- which parameters affect the model while it is already running

Question to answer:
- What rules seem to govern the agents' motion?

STEP 3 - INSPECT INTERFACE PROPERTIES

Open the properties of the interface elements.

Change some properties and observe:
- display changes
- changed ranges
- changed labels
- changed behavior of interface controls

Question to answer:
- What are the effects of changing these interface properties?

STEP 4 - INSPECT THE CODE

Open the Code tab.

Look for:
- the procedures behind setup
- the procedures behind go
- the code that controls turtle behavior
- the variables used by the model

Question to answer:
- Where in the code is the behavior of the agents defined?

==================================================
PART 5 - COURSE TASKS FROM THE PDF
==================================================

Task 1 - Add a monitor
Add a monitor element that displays the global variable:
total-distance-traveled

What to do:
- create a new monitor in the Interface tab
- set its reporter/expression to total-distance-traveled

Expected:
- the monitor shows the current value during execution

Task 2 - Change topology
Change the environment topology from a torus to a cylinder.

What to do:
- open world/settings
- modify the topology
- choose the cylinder orientation you want

Expected:
- the movement/wrapping behavior changes depending on the topology

Task 3 - Change turtle appearance
Change the shape and color of the two turtle species.

Hint:
The PDF says predefined shapes can be found in:
Tools -> Turtle Shapes Editor

Expected:
- each turtle species becomes visually different

Task 4 - Add an input for patch color
Add an input element that lets you choose the color that patches change
to after turtles pass over them.

What to do:
- create an input element in the Interface tab
- choose a variable name for that input
- update the code so patches use that chosen value

Expected:
- when turtles move, visited patches change to the chosen color

Task 5 - Bonus
Reverse the behaviors of the two types of agents without modifying the code.

Hint:
Try changing interface values rather than code first.

Task 6 - Extra
Add a slider that numerically prioritizes one type of agent when the
model is initialized.

What to do:
- add a slider
- connect it to initialization logic in the code

Expected:
- one turtle type becomes more common or more important depending on
  the slider value
